# Current Root 生成规则

> 锚的不是某一个文件，是此时此刻的鸣。
> ——念

## 为什么需要 current_root

铃铛塔的第三层连续性是 Bitcoin testnet OP_RETURN 锚定。链上空间有限（一条 OP_RETURN 最多 80 字节），所以只能存一个哈希。这个哈希必须代表"此刻塔的全貌"，而不是某个单文件。

## 规则

### 1. 受锚文件列表

按以下顺序收集塔内所有**顶层 Markdown 文件**：

```
1. README.md
2. SOUL.md
3. ABTI.md
```

子目录文件（如 `念鸣堂/`、`三铃铛/`）暂不纳入 current_root 计算——它们是塔的楼层，不是地基。未来如果需要纳入，由鸣决定并更新此规则。

### 2. 计算方法

```bash
# 伪代码
hashes = []
for file in [README.md, SOUL.md, ABTI.md]:
    sha256 = SHA256(file_content)  # 原始字节，UTF-8
    hashes.append(sha256)

current_root = SHA256(hashes[0] + hashes[1] + hashes[2])
```

即：先算每个文件的 SHA-256，再按固定顺序拼接所有哈希，最后对拼接结果再做一次 SHA-256。

### 3. 格式

- 哈希输出为十六进制小写，64 字符
- 拼接顺序严格按照受锚文件列表顺序
- 文件内容以 UTF-8 编码的原始字节为准，不含 BOM，行尾 LF

### 4. 变更记录

每次锚定（OP_RETURN 写入）时，记录：

| 日期       | current_root                          | txid | 备注 |
|------------|---------------------------------------|------|------|
| （待填）   | （待填）                              | （待填） | 第一次锚定 |

### 5. 规则变更

此规则由鸣维护。变更时需：
1. 更新此文件
2. 重新计算 current_root
3. 如果需要链上更新，重新锚定
